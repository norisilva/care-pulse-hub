# Contributing to CarePulse Hub

First off, thank you for considering contributing to CarePulse Hub! It's people like you that make CarePulse a great tool for helping human relationships.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issues to report bugs.
- Include steps to reproduce the issue.
- Describe the expected vs actual behavior.

### Suggesting Enhancements
- Open a GitHub Issue with the "enhancement" label.
- Explain why this feature would be useful for caregivers or people in conflict-prone situations.

### Pull Requests
1. Fork the repository.
2. Create a feature branch (`git checkout -b feat/my-new-feature`).
3. Commit your changes (`git commit -m 'Add some feature'`).
4. Push to the branch (`git push origin feat/my-new-feature`).
5. Open a Pull Request.

## Code Standards
- Keep internal code comments in **English**.
- Ensure the user interface (alerts, labels) remains in **Portuguese** (for now).
- Follow the human-centered spirit: software should minimize friction, not add to it.

## AI Research
We are particularly interested in local AI implementations (Ollama, heuristic-based neutralization) that don't depend on paid API keys.
